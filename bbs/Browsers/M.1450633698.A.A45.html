標題[-Fx-] ScrapBook 1.12.16 正式版釋出
時間Mon Dec 21 01:48:14 2015
官網載點： https://addons.mozilla.org/zh-tw/firefox/addon/scrapbook-x/ 全中文介紹 Wiki： https://github.com/danny0838/firefox-scrapbook/wiki/Intro-(zh-tw) 繼上次 (1.12.0a35) 以後幾件大事報告： 1. 通過 AMO 完全審核了... 2. 大幅強化搜尋引擎的語法支援， 現在可以根據內容關鍵詞、標題、來源、評註、新增時間、修改時間等各項資訊 列出搜尋結果，也可以自訂排序。 3. 大幅強化匯出 HTML 資料列表的界面，內建 AJAX 全文搜尋引擎， 且可設定自動匯出 HTML 資料列表，省去手動操作的麻煩。 現在只要把 ScrapBook 存檔位置設在 dropbox public 資料夾裡， 在全世界有網路的地方都可以當靜態網站閱覽，而且可以全文搜尋。 4. 筆記頁面功能及 HTML 編輯器強化，可在右鍵選單直接選擇要操作的功能， （雖然我自己大部分都是用熱鍵） 碎碎念： 火狐瀏覽器的前景看起來頗黯淡， 雖然我個人桌面和 Android 都是用火狐為主， 但最近將要發生的變革非常多，每一項都可能造成悲劇。 就以 ScrapBook X 來說吧， 雖然目前已經寫到 1.12.16 版，但 1.12.5 之後都還沒通過 AMO 審核及簽章， 火狐在 43 版已開始試行附加元件強制簽章，44 版以後沒簽章就不能用。 可能有人說，安全性比較重要嘛，再說不要用沒在 AMO host 的來路不明套件就好啦。 well...根據之前的經驗，新版上傳後到審核通過大概要花幾個月至半年， 這裡面很多版本是修 bug 或除蟲的， 修正版寫好後要拖好幾個月才能用，對安全會比較有保障嗎？ 總之，審核龜速的問題我沒辦法解決，想用新版又不想等簽章的， 請自行關掉 xpinstall.signatures.required 或改用舊版或特殊版火狐吧。 另外，ScrapBook (X) 是非常老的套件，自然也是用 XUL 和 XPCOM 寫成的， 不過據說火狐將要停止支援它們，且可能在 12~18 個月之內發生， 這意味著 ScrapBook X 要嘛得大幅改寫，要嘛走入歷史。 而且也不是說改寫就改寫，得先確定 ScrapBook X 需要的功能能被新 API 支援才行... 話說我之前嘗試過用 GC 寫類似套件，但 GC 似乎不支援直接寫入本地端檔案... Fx 最近越來越走向 GC，未來如果不支援，只好自求多福囉。 ScrapBook 架構陳舊，雖然現在讓它還算基本可以正常運作， 但有不少問題若要改是非全面改寫不可。 不過要用哪套程式語言寫、是否寫成哪個瀏覽器套件等等，目前還沒有結論， 在此向大家打個招呼，如有高手想跳進來搞，或許可以交流交流。 以上，有什麼想法歡迎提出。 -- ※ 發信站: 批踢踢實業坊(ptt.cc), 來自: 111.243.215.222 ※ 文章網址: https://www.ptt.cc/bbs/Browsers/M.1450633698.A.A45.html ※ 編輯: danny0838 (111.243.215.222), 12/21/2015 01:50:40 ※ danny0838:轉錄至看板 EZsoft 12/21 01:54
