DNS Server是可以設定自己內部網路的domain... 申請Domain(網域)是要讓全世界知道目前這個Domain已被註冊 說真的...我是看不太懂你在問啥... 所以我來講一下整個流程好了... 開始： 1.申請了1個網域 opoepev.idv.tw 2.該網域對應到某台自己的DNS-Server的ip 3.在DNS-Server中可以設定無限多組對應的DN-IP 如：abc.opoepev.idv.tw -> 203.123.30.50 my.opoepev.idv.tw -> 203.123.30.100 web.opoepev.idv.tw -> 203.123.30.50 注意，可以多個DN對同一個IP，但一個DN不可以對多個IP 也就是說 DN與IP的關係是 多對一 4.使用者輸入了某個屬於所申請之網域時(abc.opoepev.idv.tw) 其尋找的方式是 世界DNS->tw->idv->opoepev->abc 白話一點是：先去世界的TOP DNS找 tw對應的ip 是哪個... 進到tw的DNS後找 idv 對應的ip 又是哪個... 從idv的DNS中找 opoepev 對應的ip 在哪.... 最後問opoepev中的DNS abc要去找哪個ip? 得到 203.123.30.50，接著就會將網頁導到203.123.30.50去了!! 不知道講這樣有沒有比較了解... 申請網域和自己架DNS的差別了... ※ 引述《InitialShuk (Shuk)》之銘言： : 起先一開始在看apache virtualhost : 查了一下了解其功用 : 後來很好奇 既然架設了DNS server為什麼需要再去申請網域(ex:idv.tw)? : 目前自身解讀 : dns server可以幫自己設定申請網域的前半部 ex:*.idv.tw : 會先去tw網域找 在去idv網域 對應到本身的dns server : but top網域(tw,cn,org,com....etc)非常多 這些名稱怎麼對應到相對的網域 : 大致上就是tw為什麼對應到台灣?cn對應到中國大陸?之類的 : 是說上面還有一個終極網域?所有的domain name皆會從此開始? : 上面整串使用的詞彙不確定是否正確(剛進來...) : 覺得我問得太淺 沒自己認真翻,請推薦我概念用書 我會非常感謝orz -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From: 114.36.179.240
