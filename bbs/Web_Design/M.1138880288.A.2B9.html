※ 引述《drkkimo (Dr.K)》之銘言： : 嗨 各位好 我上學期打工的工作是幫某學校寫網頁 其實我比較專注的是在程式設計方面 : 經過上學期的打工 我覺得有一些疑問 … : 1.是否一定要作到每個瀏覽器版面都一致 ? 不一定，有的瀏覽器如 lynx 根本就只有文字介面，你是要怎樣讓它版面一致 :p 所以應該是說，每個瀏覽器都要可以正確讀取該網頁。 看得到介面的如 IE，Fx，Opera，... 至少版面不要相去太遠吧。 只有文字模式如 lynx 的，至少要能夠方便存取每個功能。 這很難嗎？ 不會阿，只要網頁寫法符合規範 (例如 XHTML 1.0 transitional)， 然後不要只會用一堆 javascript，flash，影音跟圖片， 這樣大部分的瀏覽器都能正常閱讀。 當然這是指一般情況啦，有些網頁就是要超炫超多功能， 自然就得用到 flahs 或多媒體這些，這不在一般的討論範圍。 : 2.一定要完全用DIV取代TABLE 排版嗎? 不一定。 該用 table 就用 table，該用 div 就用 div。 為什麼書上會建議用 div 取代 table，是因為以前 table 被過度濫用了。 你看看那些網頁編輯器還有以前的網頁教學書， 是不是都教你用 table 切出一個空間，讓你放入東西，或是隔出一塊空間？ 可是你仔細想想，有必要用 table 嗎？ 明明 table 就叫做表格，是應該要拿來做報表用的， 為什麼反而變成排版面的工具勒？ 所以我的建議是，當你需要做報表的時候，就用 table 吧， 其他不該用 table 的時候，就用 div 吧。 : 3.一定要符合無障礙網頁嗎？　 無障礙網頁當然是可以符合最好啦， 畢竟沒有人在嫌觀看人數太多的，不是嗎 :) 不過，不一定每個網頁都可以是無障礙， 像是那些專門表現 flahs 或是 javascript 有多棒， 或是那些相簿之類的，無障礙有什麼用？ 有些人根本就沒辦法看，你就算做了無障礙又能怎樣？ 所以，這是一種 optional。 可是無障礙有這麼難嗎？ 其實說難不難，但也不是很簡單， 不過只要照著 W3C 規範去寫網頁，大致上都符合無障礙第二級了。 (沒騙人，我之前就有試過了。我從沒打算做出無障礙，可是做完時就符合第二級了) (我只不過照著 XHTML 1.0 transitional 的遊戲規則做而已) 最後，這種東西不只是台灣才有，外國也有的。 像是 section 508 好像就是美國的無障礙吧。 (憑印象，可能有記錯) -- ╔═════════════════════════════════╗ ║ zFTPServer Suite http://zftp.twftp.org/ ║ ║完全免費的 FTP Server，功能卻是非常強大，中文支援良好，內建中文版 ║ ║『虛擬目錄』、『自動更新』、『遠端操控』、『合併多個目錄』 ║ ╚═════════════════════════════════╝ -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From: 59.116.173.179
