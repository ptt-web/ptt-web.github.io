※ 引述《LaPass (LaPass)》之銘言： : 想請問大家寫網頁的方式是哪一種？ : 1. 從無到有全部自己寫，而且用的是notepad++ : 或是editplus之類的純文字編輯器 : 2. 使用具有程式碼生成的IDE，從無到有開始寫 : 例如DreamWeaver、frontpage、visualstudio web之類的 : 3. 去找到開源的程式碼套件，例如xoops、wordpress、joomla、liferay、 : roller之類的 : 熟悉他，然後靠幾套交互使用，做出東西 : 其中包誇修改、撰寫外掛、樣板以及模組 : 4. 從無到有全部自己寫，但是會去找framework : 例如Zend、CakePHP、CodeIgniter、spring、hibernate、struts2 : 之類的來用 : 5. 先花時間建造自己的framework、套件，然後用這個framework、套件去寫東西 : 6. 用公司的framework寫東西 : 7. 其他 : 我目前是6，不過自己在寫的時候也知道，寫再多再久，也不會變成自己的 : 而且很多被包在framework底下的東西、問題..... : 以後自己獨力寫網站時還是會遇到..... : 總之，現在想累積自己獨力開發的能力 : 現在很猶豫，不知道該建造自己的framework，或是去找framework來弄熟 : 或是把幾個開源的程式碼套件弄熟 : 總而言之，現在我認為我已經「會」了，但還沒到精通的地步 : 我希望現在累積的東西，能夠讓我在未來靈光一閃的時候 : 或是一時興起想寫個有趣的功能的時候 : 能在一兩天，最多一周內，完成我一時興起想做的東西 我覺得要看個人的的背景和公司生態吧， 像我科系是網路多媒體系，進到科技公司就只是當個網頁助理， 若要說基礎，頂多就是依稀還記得大學老師教的基礎的電腦程式語言， 公司的流程大概是　 美術設計好→丟給我切版套版→丟給程式工程師套程式 一開始為了加快效率當然什麼framework都看過，但發現自己底子還不夠，就還是用自己 的方式來加快工作效率。 我也不敢稱我自己做的東是算什麼framework啦， 就只是會把常用的html tag、css抓出來罷了， 我會有一個自己做的範例資料夾， 資料夾可能會有 index.html 和css、images、js資料夾， index.html最基本的wrap、main、footer等等的html tag css除了reset外，也將一些常用的語法也早寫進去了，甚至各區的background的名稱 (header.png、dot.gif) -- 出發點也很簡單，就是『讓自己的工作流程符合公司組織上的效率』， 除了自己的工作流程外，也可以和自己的夥伴討論， 譬如說請美術設計版型的時候， 1.要記得加上hover前 hover後的效果 2.要注意有些文字是要讓資料庫跑出來，所以不能做出太特殊的效果字 3.要考慮背景在延伸的時候，延伸的背景設計會不會太突兀、留白會不會太看起來很空 叭啦叭啦 族繁不及備載~ 當然也可以引進新的framework讓同事們嚐試看看， 合的就用、不合就不要勉強，畢竟有些framework雖然真的很好用，但也會因為各夥伴的 工作習慣而用得很不順手。 不過版本控制系統還是得要有就是了，不然互相蓋檔案蓋來蓋去真的是災難的開端...Orz -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From: 222.157.252.200
