※ 引述《F23ko ( http://f23ko.com )》之銘言： : 給你一些學習的建議： : 其實我覺得PHP對初學的人來說有些難學 : 因為會學這個語言的人通常是想寫網頁 : 而網頁又因為種種原因，導致想要寫一個高階的網頁 : 就必需要學很多種東西才能寫一個網頁出來 : 不像其他語言，只學一種，還能寫出不錯的東西。 : 隨便列一列網頁會用到語言、語法： : html css js as SQL xml regexp php jsp c# VB..... : 一不小心還得去學c java c++ prel ruby之類的去寫CGI : ok 這還只是程式方面，剩下的美工還有PS那些工具要學 : 而且還可能會偏到SMTP、DNS、Linux、Windows Server等，系統、硬體方向去 : 打算全學嗎？ 哪有那麼嚴重，你講這樣比較像跟ＰＭ或客戶抱怨的時候講的 我們有個客戶老說他以前也寫過程式，哪有那麼難云云 我們私下都說「他以前哪需要同時用五個語言寫程式」－ [ 1:主要script(php|.NET|JSP)三選一, 2:HTML, 3: SQL, 4:js+jquery, 5:css ] 他那種古代人最多就是寫SQL的程式，哪會知道現在有多複雜 ￣皿￣ 但是所謂「有弱水三千，我只取一瓢飲」 web development的arsenal再大，總是可以從其中一小部份開始 本人剛學php的時候在一個計畫裡面當工讀生，有學長在管理Linux和Apache 所以不需要管server設定的事情 那時候我會的東東大概這樣 HTML: <head>, <meta>, <body>, <a>, <img>, <h*>, <p>, <br /> <table>, <form>, <input>, <textarea>, <select> <font>, <b>/<i>/<u> 概念: form的input陣列，GET/POST差異, GET URL組合 不含: <div>, <span>, <object> php: GET/POST變數讀取，資料庫操作(mysqli) 輸出: echo, print_r, var_dump, header, number_format() htmlspecialchars[_decode](), urlencode/urldecode() 流程控制: if, case, for, foreach, while 陣列使用: Array(), asort, 一般陣列/關聯(associative)陣列 字串操作: dot (.) 運算子, sprintf, substr, strpos, str_replace explode/implode, 單引號(')/雙引號(") sql: select, update, insert, where, join, like, group by, order by concat(), if(), count() 我認為以上這些，不包括session，是寫一個能用的網頁的minimal set 看起來沒多少，其實也真的沒多少 稍微進階一點，寫一個可以登入的網頁，就加上 php: session (或cookie) HTML: onclick=, onfocus=, <button>, class=, style=, <div>, <span> js: basics: var, if, for, php傳值到js的基本方法（很簡單，用echo） getElementById(), getElementsById() (這很重要，不要再看那些global object的過時邪說了，基本上是犯罪) DOM: .style, .value, .innerHTML, ... 等基本屬性 開始使用firebug或chrome/ie開發工具 css: selectors: tag, class (.), id (#), selector組合 基本css屬性(color, margin, padding, border, font-*) 以上這些我稱為第二階段，不用很強，但是概念要有 而且不要在這裡花太多時間，因為馬上要前進到第三階段 1: 選一個framework （如CodeIgniter, CakePHP, Zend Framework） 開始習慣framework的寫法和物件用法 2: 開始用jquery 熟悉 $() selector, 和 jquery ajax 在第二階段鑽研太久會產生慣性，不容易接受第三階段的方法。
