※ 引述《rehoboth (火流星！復活！)》之銘言： : ※ 引述《dale (冰城城主)》之銘言： : : 審核制、試閱制也沒辦法針對板主的壓力提出解決辦法， : : 頂多只是把小組長抓來承擔原本屬於板主的壓力（好可憐）。 : : 所以在這裡建議新的制度（在此感謝 HIbaby 那篇給我的靈感）。 : : 首先，取消連署機，AboutNew 改成正常看板。 : : 然後無論是連署新板、連署板主、罷免板主、申請廢板…… : : 一、使用者於 AboutNew 提出申請（格式可以考慮 HIbaby 那篇，但字數不用那麼多 ꄊ: : 二、討論（主要讓小組內現任板主和申請人有機會交流），為期一週。 : : 三、小組內現任板主與小組長投票。 : : 這個制度的好處是什麼？ : 這個制度的壞處遠大於好處 : : 一、不會發生多數暴力，批踢踢站民藍綠比例根本沒關係。「大不大聲」 : : 是看有沒有道理，而不是看有多少個 IP。就算一大票人跟你持相反意見， : : 就算對方烙一堆同學來投票，只要你的思維夠清楚，立場站得住腳， : : 就有可能會通過。 : 這使得「少數暴力」更容易達成 : 現任板主的負荷將會更重 : 投票意願會更低 : 每個板主都要在管理自己板板務之餘，思考別的板板務 : 才能夠使「連署新板、連署板主、罷免板主、申請廢板」這些事項進行 : 這使每個板主的工作量更大 : 板主的權力也更大 : 有心人士聯手勾結的可能也更高 : 「人數」、「知識」、「經驗」這三者 : 如果能在投票中用加權的方式處理，可能是比較好的參考 其實多數少數這些用語是沒有意義的，重點在投票權人到底是誰， 而所謂的加權，人數也就算了，知識跟經驗要怎麼加權？誰來判斷？ 我是覺得把投票權人限制在組內板主群有點怪怪的， 依現行組規，板主的產生有兩條路， 一是自行發起連署，二是由該板現任板主共同推舉， （各板自行辦理投票後產生的板主在小組看來走的其實是第二條路） 近來除了無人管理的板外，走第一條路的比較少見， 然而這始終是板主產生的最初源頭，一個板的成立靠的是連署， 當一個板沒有板主的時候也是要用連署的方式產生第一個板主， 因此無論如何，板與板主的產生最初是由使用者投票的方式產生的， 這相當容易理解，板的存在是因為有使用者使用的需求， 板主的存在是為了管理並維護板，以滿足使用者的使用需求， 到底有沒有這樣的使用需求，及該板主人選是否能滿足使用者對該看板的期待， 在現行制度下是交由使用者來判斷的； 因此相對的，當罷免連署的時候，同樣應該是由使用者來投票，來下判斷。 制度上板主的去職還存在另一個可能，那就是由小組長甚至站方決定將之去職， 不過這應該是另一回事。當然， 我們也可以想像dale的提案是把原來小組長就可以做的決定變成讓組內管理者集體決策， 不過這跟罷免連署應該就扯不上關係，這可能跟原提案人的本意不符， 而且這個方法有可能會讓小組長綁手綁腳，而且就如rehoboth所說， 各板板主管自己的板都不一定有空了，期待他們沒事還要關心一下其他板的板務， 似乎只會造成各板主更重的負擔，而這個制度的實際效果， 現行制度下看板試閱投票的投票率或可見一斑。 至於有心人士聯手勾結一事，似乎是想太多了。 : : 三、那些人邏輯不清楚，那些人說話有條有理──時間久了都可以看出來。 : : 時間可以很自然地淘汰小白。 : 一週的時間並不見得夠 : 哪些人邏輯不清楚，哪些人說話有條有理，時間久都可以看出來？ : 那為什麼U黨可以弄成那樣？ : : 四、能夠更準確地表達「民意」。目前對於「拉人來連署」這個問題 : : 只能靠著「拉更多人來連署」來解決。在新制度下，要表達民意， : : 只要在一週內把自己的想法寫出來就行了。 : 把自己的看法寫出來，也與「廢除連署機」沒有相關 此外，參照H板友及該討論串其他板友的建議，個人有點小小意見敘述如下。 我認為開放一個空間去寫連署理由或許是可行的， 但是否有必要尚待斟酌，老實講， 對立的兩造通常積怨已久，即使不是，也往往是把筆戰的戰場一換再換而已， 五百字的限制根本不是問題，對這些在政治類看板討生活的人來說， 而且字數跟理不理性沒有什麼太大的關聯， 我想大家在這個群組都待這麼久了，應該很理解這個道理。 這種規定最有可能造成的後果，就是累死小組長以下的管理者，並造成系統的負荷， 想想，以目前投票的規模，若是有一百人投票， 就要看一百篇五百字以上的東西，內容有沒有還不一定， 光只是看就算了，以前投票只要審ID上站次數跟文章數， 最多審一下一行為限的理由是不是太過天外飛來一筆，另外抓抓IP重複的， 現在理由一下子就變成一篇小作文，審查標準還很模糊， 這不是減少紛爭，而是製造紛爭，每一篇理由究竟成不成立都可能引發新的爭訟。 而且，這個制度頂多使拉票的難度提高一點， 卻不能真正阻止拉票或多重ID的存在，相比之下，很有可能得不償失。 其實問題的重心是在究竟允不允許多重ID或是拉票， 多重ID是不允許的，很清楚，問題只在執行上如何抓出來； （技術問題很多人比我在行，就不多談了。） 拉票的話，制度上並沒有不允許，且真正造成問題的不是拉票的行為本身， 有投票就有拉票的行為，這是天經地義的， 而是PTT的連署並沒有分地區投票這回事， 公開看板的板主跑回跟該看板不相關的地方請大家支持他， 其結果就是造成幽靈人口大舉移動，變成桃園縣民選新竹市長的這種情形， （純舉例而已，勿作太多聯想）所以大家才會跳腳。 那解決的方法呢？很遺憾，目前想不出來，而且似乎也沒有那麼非解決不可。 PTT的看板原則上是自由來去的，每個使用者都是每個公開看板的潛在使用者， 你無法像現實生活中那樣，用戶籍登記來劃分你可以參與那個部分的公共事務， 這就是網路世界的流動性，沒有辦法。 （因此，扁板不能說只有泛綠的才是板友，國民黨板也不能說只有泛藍的才是板友， 因為這是依議題劃分的公開看板，不是團體板，也不是班板，亦非偶像fans板。 基督教板永遠要面對對其教義的質疑，國民黨板也必須面對對國民黨的質疑， 當然，你可以叫他先看過精華區或做過功課後再發言XD。） 惟一可堪告慰的是，流動性至少是公平的， 每個人都可以靠自己的人緣想辦法去到處拉票，武器平等，呵。 至於拉票的實際效果，我想自己有拉過票的人心裡有數，並不真的那麼有效率。 更何況，以板主的罷免連署來說， 若板主已經到了要靠一個人的人緣一次又一次去對抗相對多數的板友， 我想誰比較不利應該很清楚。 然而若真的去算算過往到目前為止板主被這個機制拉下來的成功率， 似乎數字是站在板主這一邊的，既然如此， 或許現狀還在可以接受的範圍內吧。 -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From:
