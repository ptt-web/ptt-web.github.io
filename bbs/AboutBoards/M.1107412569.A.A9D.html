※ 引述《okcool (Susan說)》之銘言： : ※ 引述《dale (冰城城主)》之銘言： : : 那我來拋個磚頭吧，如果有不合適、沒思考清楚的地方， : : 還請各位多多指教。 : : 如同板主有權力、責任維持看板風氣， : : 小組長也有權力、責任維持小組各項機制的使用風氣。 : : 如果在提出申訴之前，多一些合理的檢驗機制，例如： : : 一、申訴者必須提出和板主溝通過的完整證據（當然不包括寫信亂罵一通）。 : : 二、申訴者必須提出曾在看板發言之證據。 : : 三、申訴以一篇為限，並必須符合格式。 : : （以上三點只是舉例，我的目的在於建議這個架構。） : : 然後懲罰屢犯者。 : : 像第一次犯規則警告，警告後再犯則半年內不得申訴之類的。 : 有沒有人願意一起加入討論的 : 屆時可能選擇一項較能廣為大家接受的方式 : 由現任的板主群來投票是否同意實施 : 也能提供給下一屆小組長不錯的建議 小組長同時也是組務板的板主，組務板是一般使用者上訴的地方， 如果有人濫用申訴，形同濫用組務板， 就由小組長直接將其在組務板浸水桶，也等於剝奪其往後上訴的機會， 二者合一，且理由相關，也不必多想其他的處分方法。 至於對小組長的這個處分不服，當然也是只有上訴站方一途。 問題只在於，怎樣叫做濫訴，又怎樣才能使訴訟更有效率。 其實現代的法律體系已經指出一條明路，就是要注重形式， 不符合形式的一律以程序理由駁回。省腦力也省大家時間。 使用訴訟制度的人，有義務要了解這套體系的運作模式， 在現實社會中有律師可以幫忙，BBS上當然不可能做到這一點， 不過相對的BBS上法規的技術也沒有難到真的那麼專業，所以平衡起來也還好。 像dale所提的規定就蠻可以考慮的， 首先是申訴必須符合一定格式，申訴對象、案由、證據、理由都是必要的， 由於這裡形同二審，因此必須檢附一審的判決文或其所在， 不過與板主溝通的內容如為私人信件，則不適合在此公布， 只要釋明曾經通信溝通無效即可。 不符合格式的一律駁回。這樣至少可以把爭點集中，節省大家的腦力與時間。 其次，這裡既然是要做案件的審理，就不是讓大家在這裡吵架用， 因此或可規定針對個案申訴的部分凡發文的受話者皆為審理者即小組長， 禁止兩造甚至第三人在此互相對話，違者一律砍除文章， 履犯不聽勸告者水桶之類的。 不過組務板既然叫做組務板，就有除了個案申訴以外的文章， 那個部分應該不受此限制。換言之，如果是討論抽象的規則或是組務的發展等， 應該允許大家多互相交換意見。 最後的問題可能就在於，允不允許第三人就個案申訴表示意見， （請注意在前述的邏輯之下這裡指的當然是以小組長為受話人的表示意見） 我是覺得適度的允許對小組長的審理還是有幫助的， 畢竟BBS跟現實的法庭仍舊有點不同， 有時候小組長對案件的情況沒辦法馬上了解，而兩造忙著互相指控， 如果有好心的第三人跳出來把情形講得比較情楚，或許可以節省小組長的時間。 當然，若是因此陷入大亂鬥的情形則小組長可以隨時制止，自不待言。 以上一點淺見。 -- ※ 發信站: 批踢踢實業坊(ptt.cc) ◆ From: 218.166.44.106
